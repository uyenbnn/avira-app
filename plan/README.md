# Plan Artifacts

This folder stores human-readable process plans generated by the Product Owner step in orchestration flows.

## Naming Convention

Use:

- `po-plan-<YYYYMMDD>-<short-topic>.md`

Example:

- `po-plan-20260410-tenant-auth-flow.md`

## Required Sections

Each plan should include:

1. Scope and assumptions
2. Selected SDLC phases
3. Artifact handoffs
4. Validation approach
5. Progress/status updates
